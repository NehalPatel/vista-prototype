Instruction File for AI Editor – Face Detection & Recognition Prototype

Overview
- Goal: Build a local, file-based face pipeline (no DB, no API) that detects faces with RetinaFace, aligns them, generates ArcFace embeddings using InsightFace, performs simple recognition via cosine similarity against a local known_faces set, and saves results as JSON with crops and embedding files.
- Input: Frames from previously processed videos (e.g., `vista-prototype/frames/`), or extract frames if missing.
- Outputs: Cropped face images, per-face embedding files, consolidated JSON with detections + embeddings + optional recognition results.

Folder Structure
- Base (existing): `e:/Vista Prototype/`
- New face pipeline folders (under `vista-prototype/`):
  - `vista-prototype/faces/`
    - `crops/`              # cropped faces from frames
    - `embeddings/`         # per-face `.npy` or `.json` embedding files
  - `vista-prototype/face_results/`
    - `faces.json`          # detection + landmarks + embeddings
    - `recognition.json`    # optional matching results
  - `vista-prototype/known_faces/` (optional, manual)
    - `embeddings/`         # precomputed known face embeddings (`.npy`)
    - `labels.json`         # mapping from embedding filename → person label

Dependencies
- Python 3.9+ recommended
- Core libraries:
  - `insightface`           # ArcFace embeddings (and alignment utils)
  - `onnxruntime-gpu`       # GPU runtime for InsightFace (use `onnxruntime` for CPU)
  - `opencv-python`         # image I/O and preprocessing
  - `numpy`                 # arrays, embeddings
  - `scikit-image`          # similarity transform for alignment
  - `tqdm`                  # progress bars
  - `scipy` (optional)      # cosine similarity (`scipy.spatial.distance`)
- RetinaFace options (choose one):
  1) `retinaface` (pip) – may require `mxnet`; fast but heavier to set up on Windows.
     - GPU: `mxnet-cu118` (version may vary), or CPU: `mxnet==1.9.*`
     - Note: Windows GPU builds for MXNet can be finicky; if installation is painful, use option 2.
  2) InsightFace with detector + landmarks – if RetinaFace installation is problematic, temporarily use `insightface.app.FaceAnalysis` with SCRFD detector. Interface will remain compatible so swapping to RetinaFace later is easy.

Install Commands (Windows PowerShell)
- GPU (recommended on GTX 3050):
  - `venv\Scripts\pip.exe install insightface onnxruntime-gpu opencv-python numpy scikit-image tqdm`
  - If using pip RetinaFace: `venv\Scripts\pip.exe install retinaface mxnet-cu118` (verify CUDA version)
- CPU (fallback):
  - `venv\Scripts\pip.exe install insightface onnxruntime opencv-python numpy scikit-image tqdm`
  - If using pip RetinaFace CPU: `venv\Scripts\pip.exe install retinaface mxnet==1.9.1`

Configuration Parameters
- `detect_conf`: float in [0,1], default 0.8 – minimum detection confidence to keep face.
- `similarity_thresh`: float in [0,1], default 0.6 – cosine distance threshold for “same person”.
- `maybe_range`: [0.6, 0.8] – distance considered “maybe same person”.
- `frames_dir`: path, default `vista-prototype/frames/` – input frames.
- `faces_dir`: path, default `vista-prototype/faces/` – output root.
- `results_dir`: path, default `vista-prototype/face_results/` – consolidated JSON outputs.
- `device`: `cuda` or `cpu`; auto-detect with PyTorch or ONNXRuntime.

JSON Schemas
- `face_results/faces.json` (detection + embeddings)
  {
    "frame_0001.jpg": [
      {
        "bbox": [x1, y1, x2, y2],
        "landmarks": {"left_eye": [x,y], "right_eye": [x,y], "nose": [x,y], "mouth_left": [x,y], "mouth_right": [x,y]},
        "confidence": 0.93,
        "angle": {"roll": r, "pitch": p, "yaw": y},
        "aligned": true,
        "face_crop": "vista-prototype/faces/crops/face_0001_01.jpg",
        "embedding_file": "vista-prototype/faces/embeddings/face_0001_01.npy"
      }
    ]
  }

- `face_results/recognition.json` (when matching against known faces)
  {
    "frame_0001.jpg": [
      {
        "face_crop": "vista-prototype/faces/crops/face_0001_01.jpg",
        "embedding_file": "vista-prototype/faces/embeddings/face_0001_01.npy",
        "recognized_as": "Nehal Patel",   // or "Unknown"
        "distance": 0.57,
        "confidence": 0.87                  // optional, derived from distance
      }
    ]
  }

High-Level Pipeline Flow
1) Load frames from `frames_dir`. If empty, optionally extract frames from a given video using existing pipeline.
2) For each frame:
   - Run RetinaFace detection (or detector fallback) to get `bbox`, `confidence`, `landmarks`, angle.
   - Filter detections by `detect_conf`.
   - Crop face regions and save to `faces/crops/` with deterministic naming: `face_{frameIndex}_{detIndex}.jpg`.
   - Align faces to frontal view using 5-point landmarks (similarity transform).
3) Embeddings:
   - Load ArcFace model via InsightFace and generate 512-d embeddings for each cropped/aligned face.
   - Save embedding to `.npy` in `faces/embeddings/` with same base name as crop.
   - Append record to `faces.json` under the corresponding frame.
4) Recognition (optional, file-based):
   - Load known face embeddings from `known_faces/embeddings/` and label mapping from `known_faces/labels.json`.
   - Compute cosine distance against each known embedding; assign label with minimum distance if `distance < similarity_thresh`, else mark Unknown; mark Maybe if in [0.6,0.8].
   - Save results to `recognition.json`.

Implementation Plan (Modules & Responsibilities)
- `face_pipeline/paths.py`
  - Provide canonical paths: `FRAMES_DIR`, `FACES_DIR`, `CROPS_DIR`, `EMBED_DIR`, `FACE_RESULTS_DIR`, `KNOWN_FACES_DIR`.
  - Ensure directories exist; create on first run.

- `face_pipeline/detection.py`
  - `load_detector(device)`: initialize RetinaFace (preferred) or fallback detector.
  - `detect_faces(detector, frame_bgr, conf_thresh) -> List[Det]` where `Det` contains bbox, confidence, landmarks, angle.
  - `crop_face(frame_bgr, bbox) -> crop_bgr` with bounds checking.
  - `align_face(crop_bgr, landmarks) -> aligned_bgr` using 5-point similarity transform.
  - Save crops to `CROPS_DIR` with predictable names.

- `face_pipeline/embeddings.py`
  - `load_arcface(device)`: initialize InsightFace embedding model.
  - `compute_embedding(model, aligned_bgr) -> np.ndarray(shape=(512,))`.
  - Save `.npy` embeddings to `EMBED_DIR`.

- `face_pipeline/recognition.py`
  - `load_known_embeddings(dir) -> List[(embedding, label)]` from `known_faces/embeddings/` and `labels.json`.
  - `cosine_distance(a, b) -> float` (use numpy or scipy).
  - `match(embedding, known, thresholds) -> {label, distance, confidence}`.
  - Write consolidated `recognition.json` per frame.

- `face_pipeline/run.py` (CLI)
  - Args: `--frames-dir`, `--detect-conf`, `--device`, `--do-recognition`, `--known-faces-dir`.
  - Or `--video` to auto-extract frames via existing pipeline if frames don’t exist.
  - Orchestrate: detection → alignment → embeddings → optional recognition → write JSONs.
  - Deterministic naming and overwrite protection (skip existing crops/embeds unless `--force`).

Detector Notes (RetinaFace vs Fallback)
- Preferred: RetinaFace for best accuracy with occlusion, side faces, masks.
- Practical fallback: `insightface.app.FaceAnalysis` with SCRFD detector if RetinaFace + MXNet is problematic on Windows.
- Keep detector interface thin (`detect_faces`) so swapping is trivial.

Alignment Notes
- Use the 5-point landmarks (eyes, nose, mouth corners) to compute a similarity transform to a canonical template.
- Libraries: `skimage.transform.SimilarityTransform` or custom OpenCV affine with reference points.
- ArcFace benefits significantly from aligned inputs; set `aligned=true` when applied.

Embedding Model
- Use InsightFace ArcFace model via `insightface` with ONNXRuntime.
- GPU preferred (`onnxruntime-gpu`); CPU works for prototypes but slower.
- Output vector: 512-d float32; save as `.npy`.

Recognition Logic (File-Based)
- Known faces folder contains `.npy` embeddings and a `labels.json` mapping.
- Compute cosine distance between detected face embedding and each known embedding.
- Thresholds:
  - `distance < 0.6` → same person
  - `0.6 ≤ distance < 0.8` → maybe
  - `≥ 0.8` → different person/unknown
- Confidence can be derived as `1 - min(distance, 1.0)` for a simple proxy.

Naming & Persistence
- Crops: `face_{frameIndex}_{detIndex}.jpg`
- Embeddings: `face_{frameIndex}_{detIndex}.npy`
- JSON files written atomically; if partial run, resume by skipping already existing files unless `--force`.

Performance & Batch Tips
- Batch detection per frame; embeddings can batch-process multiple faces.
- Cache model instances; don’t reinitialize inside loops.
- If GPU VRAM is limited, reduce batch size or process sequentially.

Error Handling
- Skip frames with read errors; log and continue.
- Bounds check bboxes; clamp to image dimensions.
- If landmarks missing or malformed, skip alignment and mark `aligned=false`.

Testing & Validation
- Smoke test with 10–20 frames.
- Validate embeddings dimensionality and reasonable magnitude (L2 norm ~1 if normalized).
- Confirm JSON structure matches the schemas above.
